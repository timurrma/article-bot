# Article Bot — Требования

## Что делает бот

Телеграм-бот для одного пользователя (личный). Каждый день в 09:30 по Москве присылает образовательную выжимку из статьи или книги. Материал берётся по ссылке — бот сам ходит и читает. Также умеет отвечать на вопросы пользователя на основе знаний GPT-5 mini.

---

## Источник контента

- Пользователь кидает любую ссылку через `/add <ссылка>`
- Бот определяет тип источника и читает текст соответствующим способом
- Текст **не хранится** — бот ходит по ссылке каждый раз при отправке, сохраняется только ссылка и offset (до какого символа дочитали)
- Поддерживается очередь из нескольких документов

### Поддерживаемые типы источников

| Тип | Определение | Как читать |
|---|---|---|
| Google Docs | `docs.google.com/document/` | Google Docs API |
| Google Drive PDF | `drive.google.com` + mime-type PDF | Google Drive API → PyMuPDF |
| Прямая ссылка на PDF | URL заканчивается на `.pdf` | Скачать → PyMuPDF |
| Habr / Medium / любой URL | всё остальное | HTTP + BeautifulSoup (извлечь основной текст) |

### Если ссылка недоступна или тип не распознан
- Бот пробует прочитать как обычную веб-страницу
- Если не получилось — сообщает об ошибке и не добавляет в очередь

---

## Разбивка материала

- **Чанк = ~750 слов** (≈ 5 минут чтения)
- Чанк **никогда не обрывается на середине абзаца** — после набора ~750 слов бот добивает до конца ближайшего абзаца
- Если есть явные заголовки глав — не разрезает главу пополам, если она влезает в лимит ±30%
- Прогресс хранится как номер символа (offset) в документе

---

## Формат выжимки

Выжимка должна быть **образовательной и полноценной**, не просто пересказ. Структура сообщения:

```
📖 [Название документа] — часть N/M

─────────────────────
📝 О ЧЁМ ЭТОТ КУСОК
─────────────────────
[2–3 предложения: контекст и главная мысль раздела]

─────────────────────
💡 КЛЮЧЕВЫЕ ИДЕИ
─────────────────────
• [тезис 1]
• [тезис 2]
• [тезис 3]
...

─────────────────────
🗣 ЦИТАТЫ
─────────────────────
«[цитата из текста]»
«[цитата из текста]»

─────────────────────
🧠 ЧТО СТОИТ ЗАПОМНИТЬ
─────────────────────
[1–2 предложения: главный вывод или инсайт]
```

Промпт для LLM явно требует: глубокий анализ, нетривиальные идеи, не поверхностный пересказ.

---

## Расписание

- Одна выжимка в день
- Время по умолчанию: **09:30 Europe/Moscow**
- Настраивается через `/settings time HH:MM`

## Альтернативные сценарии

### Документ закончился
1. Бот присылает сообщение: _"✅ «[Название]» прочитан полностью! Следующий в очереди: «[Название 2]»"_
2. Автоматически переходит к следующему документу в очереди
3. На следующий день — первый чанк из нового документа

### Очередь пуста
- Бот не присылает ничего в этот день (не спамит)
- Один раз присылает напоминание: _"📭 Очередь пуста. Добавь новый материал через /add"_ — только в момент когда закончился последний документ, не каждый день

### Источник недоступен (ошибка чтения)
- Бот присылает: _"⚠️ Не могу прочитать «[ссылка]». Проверь доступ и попробуй /now"_
- Прогресс не сдвигается, завтра попробует снова

### Источник изменился (стал короче)
- Если offset превышает длину текста — бот присылает: _"⚠️ «[Название]» стал короче чем был. Начать сначала? /restart_doc или завершить /finish_doc"_

### Пользователь присылает /add с уже существующей ссылкой
- Бот проверяет по URL: _"Этот материал уже есть в очереди (позиция N). Добавить повторно?"_ — inline кнопки Да / Нет

### Пользователь присылает не ссылку
- Бот отвечает: _"❌ Нужна ссылка (Google Docs, PDF, статья). Попробуй ещё раз."_

### Claude API недоступен
- Бот присылает: _"⚠️ Не смог сгенерировать выжимку (ошибка AI). Попробую снова через /now или завтра по расписанию."_
- Прогресс не сдвигается

### Пользователь хочет перечитать уже пройденное
- `/restart_doc` — сбросить прогресс текущего документа на начало
- `/restart_doc all` — сбросить все документы

## Команды бота

| Команда | Действие |
|---|---|
| `/add <ссылка>` | Добавить Google Doc в очередь |
| `/queue` | Показать очередь и текущий прогресс по каждому документу |
| `/skip` | Пропустить текущий чанк (сдвинуть прогресс вперёд без выжимки) |
| `/pause` | Поставить на паузу (бот не присылает по расписанию) |
| `/resume` | Возобновить |
| `/settings time HH:MM` | Изменить время доставки |
| `/now` | Прислать выжимку прямо сейчас |
| `/restart_doc` | Сбросить прогресс текущего документа на начало |
| `/remove` | Удалить документ из очереди |

---

## Режим вопросов и ответов

- Любое сообщение вне команд — вопрос к боту
- Бот отвечает на основе знаний GPT-5 mini (не RAG по документу)
- В системный промпт подкидывается контекст: название текущего документа и тема, чтобы ответы были в тему

---

## Стек

| Компонент | Решение |
|---|---|
| Язык | Python 3.11+ |
| Telegram | `aiogram` 3.x |
| LLM | OpenAI API (gpt-5-mini) |
| Чтение источников | `google-api-python-client`, `PyMuPDF`, `BeautifulSoup4`, `httpx` |
| Хранилище | SQLite (`aiosqlite`) в Railway Volume `/data/bot.db` |
| Планировщик | `APScheduler` |
| Деплой | Railway |
| Конфиг | `.env` |

---

## Деплой

- **Railway** — один сервис, всегда онлайн
- БД: SQLite в persistent Volume → `/data/bot.db`
- Env vars: `BOT_TOKEN`, `OPENAI_API_KEY`, `GOOGLE_CREDENTIALS_JSON`
- Google Docs/Drive: сервисный аккаунт Google, credentials в JSON (в env var) — нужен только если используются Google-ссылки

---

## Ограничения / out of scope

- Нет веб-интерфейса
- Нет нескольких пользователей
- Только Google Docs (не PDF, не EPUB, не Notion)
- Нет RAG по тексту в Q&A режиме
